# Mock IT Procurement Demo Dataset

Fictional demo assets: **10 purchase-order / order-form PDFs** for IT products
(each with a different visual design) and a **SQLite procurement database**
that references those PDFs and tracks agreements with renewal dates.

Everything is generated by `generate_procurement_demo.py` (deterministic — no
randomness). Regenerate with:

```bash
pip install reportlab
python3 generate_procurement_demo.py
```

## Database schema

| Table | Purpose |
|---|---|
| `vendors` | 10 IT vendors (contact, address, category) |
| `agreements` | Contracts per vendor: type, term, **renewal_date**, auto_renew, notice days, annual value, status |
| `purchase_orders` | 10 POs — totals, status, department, and **`pdf_path`** pointing at the PDF file |
| `po_line_items` | 38 line items (SKU, qty, unit price) that sum to each PO's subtotal |
| `invoices` | 10 invoices against the POs (paid / pending / overdue, some split-billed) |
| `documents` | Generic file registry linking every PDF to its entity (`entity_type` + `entity_id`) |

Views: `upcoming_renewals` (agreements sorted by renewal date) and
`po_summary` (PO totals vs. amounts paid, with PDF path).

`pdf_path` / `file_path` values are relative to this directory
(e.g. `pdfs/PO-2025-0841_Dell.pdf`).

## Sample queries

```sql
-- Renewals coming up, with what we spend annually
SELECT * FROM upcoming_renewals WHERE renewal_date <= '2026-12-31';

-- Open (unpaid) exposure per vendor
SELECT vendor, SUM(total - paid_amount) AS outstanding
FROM po_summary GROUP BY vendor HAVING outstanding > 0;

-- Overdue invoices with a link to the source PO document
SELECT i.invoice_number, v.name, i.amount, i.due_date, po.pdf_path
FROM invoices i
JOIN purchase_orders po ON po.id = i.po_id
JOIN vendors v ON v.id = i.vendor_id
WHERE i.status = 'overdue';
```

All companies, people, prices, and documents are fictional and for demo use only.
